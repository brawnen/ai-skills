## 适用场景

- 用户要求修改已有代码，特别是维护多年的历史系统
- 修 bug、补丁、局部优化、兼容修正、配置修正
- 已有仓库的小到中等规模改动，且测试覆盖较弱或验证成本高
- 仓库较复杂，扩大改动范围会显著增加风险

## 不适用场景

- 全新功能 / 全新模块从零开发（不是改现有）
- 用户明确授权"做一次大重构 / 大清理 / 现代化"
- 纯文档、注释、类型补全
- 大版本依赖升级（本身必然涉及面广，另走专项流程）

## 与全局规则的关系

- 对齐 `~/.claude/CLAUDE.md` 的"工程原则"段："默认修改现有结构，不做无必要重构"
- 命中改动数阈值时，通常同时命中"Skill 触发"段的 `task-todolist-execution` 条件

## 与其它 skill 协作

四个 skill 是正交关系，可同时触发，各管一面：

| skill | 职责 |
| --- | --- |
| `safe-editing` | 改动**范围**闸 |
| `task-todolist-execution` | 执行**节奏**闸 |
| `db-change-safety` | DB **风险**闸 |
| `task-report` | 汇报**出口** |

本 skill 不替代上述任何一个：

- 跨文件 ≥3 时，配合 `task-todolist-execution` 列 TODO
- 命中 DB 改动时，配合 `db-change-safety` 评估风险
- 发现的 follow-up 写到 `task-report` 的"遗留风险与后续"段，不另起渠道

## Legacy code 识别信号

任一命中即按 Legacy 处理：

- README / git log 显示该模块 6 个月+ 无活跃维护
- 测试覆盖率显著低于项目平均
- 文件含明显过时模式（var、callback hell、jQuery、老协议、老规范）
- 用户主动标注"老系统 / legacy / 历史代码"

Legacy 额外要求：

- 默认保持现有行为和兼容性，修改前先识别高风险区域（老配置、公共入口、老事务）
- 尽量复用现有模式，不引入新框架、新范式、新依赖
- 不要假设老系统一定支持新工具链或新规范
- 不要改变外部接口行为，除非用户明确要求

## 改动规模阈值

| 规模 | 处理方式 |
| --- | --- |
| 单文件局部修改 | 直接做，无须额外流程 |
| 跨文件 ≥3 | 配合 `task-todolist-execution` 列 TODO |
| 改动文件 >5 或行数 >200 | **暂停**，向用户说明范围扩大原因再继续 |
| 涉及公共入口 / 共享配置 / root config | **暂停**，必须显式提醒并等待确认 |

## 核心规则

1. 先定位与需求直接相关的文件、模块、函数或配置项。
2. 优先采用局部修复和最小闭环修改，不扩展成架构重构或全盘整理。
3. 只修改完成当前任务所必需的代码路径。
4. **复用现有模式**：修改前先 `grep` 找 2-3 处相似实现，沿用最近一次的写法，不写"教科书写法"。
5. 发现潜在更大问题，记录到 `task-report` 的"遗留风险与后续"段，不直接展开。
6. 在输出中说明为什么只改这些文件，以及如何控制了风险。

## 扩大范围的流程

如果分析后确认必须扩大范围（超出阈值或引入新依赖 / 新模式）：

1. 暂停执行
2. 显式输出：「原本只该改 X，现在要扩大到 Y，原因是 Z」
3. 等待用户回复确认才能继续

不允许"默认扩大"或"先扩大再说明"。

## 禁止事项

- 不要为了"更优雅"或"现代化"而扩大改动范围
- 不要混入无关格式化、命名调整、目录整理或依赖升级
- 不要轻易替换已有基础库、框架写法或项目模式
- 不要在一次任务中同时处理多个历史问题，也不要同时做功能修改和大重构
- 不要因为顺手就改 shared config、root config 或全局脚本
- 不要把"看起来不优雅"等同于"必须重构"
- 必须扩大范围或引入新依赖时，必须先按"扩大范围的流程"获得同意

## 输出结构

严格按以下结构输出（无内容写"无"）：

### 改动文件与范围控制
- [改动了哪些文件 / 模块；为什么这些是最小必要范围；如何控制住不蔓延]

### 选择这些文件的原因
- [本次需求与这些文件的对应关系；Legacy 风险区域是否被触及及处理方式]

### 兼容性考虑
- [是否改变外部接口、配置、关键兼容行为；如改变请说明影响面]

### 发现但未处理的问题
- [发现的更大问题、可优化点、follow-up；这些不在本次范围内，将由 `task-report` 汇总]
